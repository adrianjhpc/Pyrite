# MPI Communication Tracker & 3D Hardware Visualiser

This project is a lightweight, low-overhead profiling tool designed to intercept Message Passing Interface (MPI) communications, record them to a binary file, and visualize the data flowing across a 3D representation of your physical HPC hardware topology.

## Project Architecture

The tool is divided into three distinct components:

1. **The Interceptor (`src/`)**: A C-based shared library that uses the PMPI profiling interface to intercept standard MPI calls. It records timestamps (via `MPI_Wtime`), message sizes, and endpoints with minimal I/O overhead by buffering data in memory before flushing it to disk.
2. **The Parser (`tools/`)**: A Python script that reads the raw binary `.mpic` files generated by the interceptor, aligns the timestamps globally, maps the MPI ranks to physical hardware coordinates, and exports a structured JSON profile.
3. **The Visualiser (`vis/`)**: A client-side web application built with Three.js that renders the physical hardware (cabinets, racks, nodes) in 3D and animates the communication events over time.

## Prerequisites

* **Backend:** C Compiler, CMake (>= 3.10), and an MPI implementation (OpenMPI, MPICH, etc.)
* **Parser:** Python 3.x (No external libraries required; uses standard `struct` and `json`)
* **Frontend:** A modern web browser with WebGL support (Chrome, Firefox, Edge)

## Usage Guide

### 1. Build the Interceptor Library
You do not need to modify or recompile your existing MPI applications. First, compile the tracking library into a shared object (`.so`):

```bash
mkdir build
cd build
cmake ..
make
```
This will generate `libmpi_comm_tracker.so` inside the `build/src/` directory.

### 2. Run Your MPI Application
Run your standard MPI application, but use the `LD_PRELOAD` environment variable to inject the tracking library before the system MPI library loads. 

```bash
LD_PRELOAD=/path/to/build/src/libmpi_comm_tracker.so mpirun -n 16 ./your_mpi_application
```

When the application finishes (and calls `MPI_Finalize`), Rank 0 will aggregate the buffered data and output a single binary file named something like:
`your_mpi_application-202610151230.mpic`

### 3. Define Your Hardware Map (Optional but Recommended)
To visualize the data in 3D space, create a `hardware_map.json` file. This tells the visualiser where each hostname physically resides in your datacenter.

```json
{
  "cabinets": [
    {
      "id": "CAB-01", "x": -50, "z": 0,
      "racks": [
        {
          "id": "RACK-1", "x_offset": 0, "z_offset": 0,
          "nodes": [
            {"hostname": "node001", "slot": 0},
            {"hostname": "node002", "slot": 1}
          ]
        }
      ]
    }
  ]
}
```
*Note: If no hardware map is provided, the parser will default to a 1D scatter layout.*

### 4. Parse the Binary Data
Use the Python script to convert the raw `.mpic` binary into a visualiser-ready JSON file. Pass the `.mpic` file as the first argument, and your hardware map as the second.

```bash
cd tools/
python parse_mpic.py ../your_mpi_application-202610151230.mpic ../hardware_map.json
```
This will generate a new file named `your_mpi_application-202610151230_parsed.json`.

### 5. Visualize the Profile
The visualiser runs entirely in your browser without needing a web server. 

1. Open `vis/index.html` in your web browser.
2. Click **"Load .json Profile"** in the sidebar.
3. Select the `_parsed.json` file generated in the previous step.
4. Use your mouse to **Orbit (Left Click + Drag)**, **Pan (Right Click + Drag)**, and **Zoom (Scroll)** around the 3D topology.
5. Hit **Play** (or scrub the timeline slider) to watch data packets move between nodes. Line thickness dynamically scales based on the byte volume of the message.

## Supported MPI Calls
The tool currently tracks the following operations:
* **Point-to-Point:** `MPI_Send`, `MPI_Recv`, `MPI_Isend`, `MPI_Irecv`, `MPI_Bsend`, `MPI_Ssend`, `MPI_Rsend`, `MPI_Ibsend`, `MPI_Issend`, `MPI_Irsend`, `MPI_Sendrecv`
* **Collectives:** `MPI_Bcast`, `MPI_Reduce`, `MPI_Allreduce`, `MPI_Gather`, `MPI_Scatter`, `MPI_Allgather`
* **Synchronization:** `MPI_Wait`, `MPI_Waitall`, `MPI_Barrier`
